terraform-aws-ec2-instance

Architecture Document — Production-Ready AWS EC2 Instance Module

Executive Summary

Production-ready Terraform module for provisioning AWS EC2 instances with launch templates, IAM instance profiles, EBS optimization, IMDSv2 enforcement, and SSM access. The module supports encrypted root and additional EBS volumes, configurable security groups with ingress rules, user data scripts, and includes a sub-module for spot instance provisioning with configurable interruption behavior. All instances default to secure configurations including private networking, mandatory IMDSv2 tokens, and encrypted storage.

Data Flow

1
Launch template defines the instance configuration: AMI, type, EBS settings, metadata options, and user data.
2
EC2 instance is provisioned from the launch template in the specified VPC subnet with private networking.
3
Security group controls inbound traffic via configurable ingress rules; egress is open by default.
4
IAM instance profile grants the instance SSM access, CloudWatch permissions, and any additional policies.
5
IMDSv2 enforcement requires session tokens for all metadata requests, preventing SSRF-based credential theft.
6
Additional EBS volumes are created and attached to the instance with independent encryption and IOPS configurations.

Security Controls

IMDSv2 Enforcement

Instance Metadata Service v2 is required by default (metadata_http_tokens = "required"), preventing SSRF attacks that exploit IMDSv1.

EBS Encryption by Default

Root and additional volumes are encrypted by default with optional customer-managed KMS keys.

Private Networking

Instances are private by default (associate_public_ip = false), requiring explicit opt-in for public IP assignment.

SSM Access (Keyless SSH)

Systems Manager access is enabled by default, eliminating the need for SSH key pairs and bastion hosts.

Least-Privilege IAM

IAM role includes only SSM and CloudWatch policies by default; additional policies must be explicitly specified.

Metadata Hop Limit

Metadata hop limit defaults to 1, preventing containers from accessing the host instance metadata service.

Industry Adaptation

Healthcare

HIPAA-ready with encrypted volumes, private subnets, IMDSv2, and no public IPs. SSM access eliminates SSH key management compliance burden.

Finance

Dedicated tenancy support for regulatory isolation. KMS-encrypted volumes for financial data. Detailed monitoring for SLA compliance metrics.

Government

GovCloud-compatible IAM policies. Restricted metadata access. Security group ingress whitelisting. Placement group support for HPC workloads.

Retail

Spot sub-module for cost-optimized batch processing. User data for automated application bootstrapping. Multiple EBS volumes for data and log separation.

SaaS

Launch template versioning for rolling deployments. Custom AMI support for golden image pipelines. Flexible security group rules for multi-tier architectures.

Production Readiness Checklist

  • Enforce IMDSv2 with metadata_http_tokens = "required"
  • Enable EBS encryption on root and all additional volumes
  • Disable public IP assignment unless explicitly required
  • Use SSM Session Manager instead of SSH key pairs
  • Attach only necessary IAM policies (least privilege)
  • Configure security group ingress rules for specific CIDR blocks only
  • Enable detailed monitoring for 1-minute CloudWatch metrics
  • Use gp3 volumes for cost-effective, high-performance storage
  • Set metadata hop limit to 1 to prevent container metadata access
  • Apply consistent tags for cost tracking and resource governance
